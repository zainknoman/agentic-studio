## What is RICE?

RICE is a feature prioritisation framework that produces a numeric score for each feature,
removing gut-feel debates and giving the team a defensible, data-driven backlog order.

**Formula:** `RICE Score = (Reach × Impact × Confidence) / Effort`

A higher RICE score means the feature delivers more value per unit of investment.

---

## Scoring guide

### Reach (users per quarter)
Estimate how many users will use this feature within 90 days of shipping.
Use absolute numbers, not percentages.

| Estimate | Example |
|---|---|
| 1 000+ | Core workflow that every user must complete |
| 500–999 | Feature used by more than half of active users |
| 100–499 | Feature used by a specific user segment |
| < 100 | Power-user feature or admin-only capability |

### Impact (per user)
How much does this feature move the primary metric for each user who encounters it?

| Score | Label | Description |
|---|---|---|
| 3 | Massive | Directly drives conversion, retention, or core value delivery |
| 2 | High | Significantly improves user experience or workflow |
| 1 | Medium | Noticeably helpful; moderate workflow improvement |
| 0.5 | Low | Minor convenience; nice to have |
| 0.25 | Minimal | Marginal improvement; edge-case benefit |

### Confidence (how sure are we?)
How confident are we in the Reach and Impact estimates?

| Score | Label | Signal |
|---|---|---|
| 100% | High | User research, validated demand, or existing data confirms it |
| 80% | Medium | Qualitative signals, analogous product data, or team consensus |
| 50% | Low | Educated guess; no direct validation yet |

### Effort (person-weeks)
Total engineering time to design, build, test, and ship this feature.
Use whole-week increments. Include frontend, backend, and QA time.
